# 识别能力方案

## 基本原则

- 项目计划同时支持浏览器本地识别和服务器识别，但实现顺序上先完成浏览器本地识别闭环，服务器识别作为后续阶段能力补齐；
- 浏览器端模型加载与推理放在 `4CImageSeg.Web.Client`，`4CImageSeg.ApiService` 负责识别相关 API、任务状态和后续服务器推理，注意避免混用；

## 本地识别实现

- 本地识别实现采用 `ONNX Runtime Web + WebGPU`；
- 使用公开预训练模型（计划使用YOLO26s COCO）完成单图导入、模型加载、单图推理、结果叠加显示和导出的最小闭环；
- 多图处理、视频逐帧识别和摄像头实时识别等高级功能，等单图流程稳定后再继续扩展；

## 模型与契约

- 训练框架可以独立选择，但部署模型优先统一为 `ONNX`，尽量让浏览器端和服务器端复用同一种模型格式；
- 统一识别结果 DTO，至少包含输入源、模式、模型版本、检测时间、类别、置信度和边界框；

## 实施顺序

1. 修正前端到 `4CImageSeg.ApiService` 的调用接线，明确页面如何稳定访问 API；
2. 定义统一 DTO 和模式枚举，固定任务提交和识别结果结构；
3. 在 `4CImageSeg.Web.Client` 中实现浏览器端单图识别闭环，并完成结果叠加显示与结果图导出；
4. 扩展多图、视频、摄像头实时识别，以及真正的服务器端识别能力；
